## 🔧 日常运维命令

> 所有 `docker compose` 命令需要在 `/vol2/1000/secret-collector` 目录下执行，或使用 `sg docker -c "..."` 以当前用户调用 Docker。

### 查看服务状态

```bash
# 标准方式（需要 docker 组权限）
cd /vol2/1000/secret-collector && docker compose ps

# 或通过 sg docker
sg docker -c "docker compose ps" --directory /vol2/1000/secret-collector
```

### 重启服务

```bash
cd /vol2/1000/secret-collector && docker compose restart [backend|frontend|postgres]
```

### 查看日志

```bash
# 后端日志
docker compose logs -f backend

# 前端日志
docker compose logs -f frontend

# 数据库日志
docker compose logs -f postgres

# 最近 50 行所有服务
docker compose logs --tail=50
```

### 更新 & 重构建

```bash
cd /vol2/1000/secret-collector && git pull && docker compose up -d --build
```

## 🔄 重试失败下载

媒体下载失败后会自动重试（指数退避：1m → 5m → 25m → 125m），最多 5 次。

如需手动重置所有失败任务：

```sql
UPDATE media SET status = 'pending', attempts = 0, last_error = NULL WHERE status = 'failed';
```

## 🚨 常见问题

### Q: 媒体下载失败怎么办？

1. 检查网络：`curl -x http://100.85.18.9:7890 -I https://x.com`
2. 查看后端日志：`docker compose logs backend | grep -i error`
3. 检查数据库中的 `last_error` 字段
4. 手动重试：`UPDATE media SET status='pending' WHERE status='failed';`

### Q: 前端打不开？

1. 检查 frontend 容器：`docker compose logs frontend`
2. 检查后端是否健康：`curl http://localhost:8080/healthz`
3. 重启前端：`docker compose restart frontend`

### Q: 数据库连接失败？

1. 等 postgres 健康检查通过：`docker compose ps`
2. 确认 `DATABASE_URL` 未变
3. 重启后端：`docker compose restart backend`

### Q: 如何修改下载代理？

编辑 `docker-compose.yaml` 中的 `HTTP_PROXY` / `HTTPS_PROXY` 环境变量，然后 `docker compose up -d --build backend`。
